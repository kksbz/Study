개인 프로젝트 때 필요한 것(포트폴리오)

1. 원본 소스코드 -> 깃 (압축한 프로젝트 원본, 빌드 파일)
2. 개발 전 기획서 -> ppt등으로 제작 후 pdf로 export
	- ui 배치도
	-구현할 주요 기능 기술 스택 리스트 업
	- 몇 개의 씬으로 구현하는지
	- 어떤 구현이 들어가는지
	- 어떤 입력장치에서 이벤트를 받는지
	- 어떤 플랫폼으로 빌드할 것인지
	- 위에서 결정한 사항을 바탕으로 개발 일정 산출

3. 게임 플레이 동영상 -> 영상편집 툴을 활용하여 제작 후 유튜브 업로드로 공유
	- 동영상의 가장 앞, 뒤에 경일게임 아카데미 인트로 동영상 삽입
	- 동영상 진행중 경일게임 아카데미 워터마크 삽입
	- 분량은 약 3분 내외

4. 개발 이후 기술문서 -> ppt등으로 제작후 pdf로 export
	- 이 게임이 대략 어떤 게임인지
	- 내가 뭘 구현 했는지
	- 플로우, 클래스 다이어그램
	- 볼륨이 어느 정도 되는지
	- 개발일지
		- 어떤 자료구조를 왜 사용했는지
		- 내가 구현하던 도중 경험한 이슈 또는 어떤 어려운 점이 있었는지
		- 이후 이슈를 어떻게 대응했는지
		- 왜 그렇게 대응했는지
	- 원본 소스코드 링크(깃)
	- 동영상 링크(유튜브)

5. 개발 일정관리(구글 캘린더 혹은 트렐로 등으로 연습할 것)
