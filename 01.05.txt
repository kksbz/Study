git branch

git barnch -r
원격 저장소에 있는 목록
git checkout 브렌치이름 < 브렌치이름으로 이동
git checkout -b 브렌치이름 <새로운 브렌치 만들기 push를해줘야 등록됨

$ git branch --set-upstream-to=origin/브렌치이름 < 브렌치이름으로 셋업스트림하는거
이제 git push origin 브렌치이름 이렇게 안써도됨

git branch -d 브렌치이름< 브랜치이름 로컬에서 지우기
git push origin --delete 브렌치이름 < 원격저장소에서 브렌치이름

git merge origin/병합할브렌치이름
현재 브렌치의파일을  origin/다른브렌치 와 병합할 때 씀
충돌이 일어나지않으면 오토 merge가 진행되어 새로운 콘솔창으로 이동됨
->i 누르면 인설트상태로 진입 안에 주석을 달수있음 esc 나가기
   :세미콜론 입력 -> 맨밑하단 글씨입력가능 wq(저장하고 나가기) 입력엔터

충돌이 일어나면 충돌난 파일명이 나옴
그파일을 비쥬얼스튜디오코드로 열면 
파일에서 살릴것과 지울것을 정할 수 있음 ->파일 수정하고 병합하면됨

병합후에는 반드시 태그를 넣어서 백업본을 만들어놔야됨 ->후에 작업시 에러상황이면 롤백하기위함

ex)
main, Develop, a, b, c ->5개의 브렌치가 있고 Develop브렌치 파일을 기본으로 설정
a, b, c 각각 Develop파일을 따로따로수정해서 각각 브렌치에 푸쉬한 상황
a, b, c 각 작업된 파일을 모두 병합->최종적으로 Develop브렌치에 병합된파일이 저장됨
(a, b, c는 병합완료후 푸쉬하고 브렌치 삭제)
main에서 Develop브렌치를 pull해서 가져오면끝 

파일넣는건 똑같음
git add .하고 커밋하고 푸쉬하기
git push origin 브렌치이름<하면 브렌치로 깃푸쉬됨 origin은 원격저장소를 뜻함

푸쉬할 때 각 브렌치에서 수행한거는 브렌치로 푸쉬해야됨

각 브렌치에서 작업된파일을 머지로 병합할 수 있음
